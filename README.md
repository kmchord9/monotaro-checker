# Monotaro Checker (モノタロウ・チェッカー)

このプロジェクトは、モノタロウのECサイトでの購入における事務的なミスをソフトウェア的に防止するためのChrome拡張機能です。

## 核心的な仕様

1. **ホワイトリスト方式（デフォルト無効）**
   - 注文ボタンや見積ボタンを検出すると即座に無効化（白背景・グレー文字）し、「判定中」状態にします。
   - すべてのビジネスルールをクリアした場合のみ、ボタンが有効化されます。

2. **パニックモード（セーフティロック）**
   - サイトの仕様変更などで必要な要素（ボタン、金額表示、支払方法など）が見つからない場合、画面全体をロックします。
   - 誤った判定で注文が進んでしまうリスクを避けるための安全装置です。

3. **バリデーションルール**
   - **金額閾値**: 小計が 3,500円（デフォルト `THRESHOLD_PRICE`）未満の場合は購入・見積不可。
   - **消費税計算**: 小数点以下の端数計算において不一致リスクがある場合は停止。
   - **支払方法**: ページBにおいて「都度払い」以外が選択されている場合は強制的に変更・固定。

---

## サイト仕様変更時のメンテナンス項目

モノタロウのサイトデザインやHTML構造が変更された場合、以下の要素を確認・修正してください。

### 1. ページ判定ロジック (`getPageType`)
- **バスケットページ**: `func` パラメータが `monotaro.basket.showListServlet.ShowListServlet` であるか。
- **注文確認ページ**: `func` パラメータが `monotaro.checkout.confirm.show_init_edit_servlet.ShowInitEditServlet` であるか。
- **見積書（印刷）**: URLに `/basket/estimate/print` が含まれているか。

### 2. 重要セレクタ一覧 (`content_script.js`)

#### ページA: バスケット
- **空判定**: `.BasketNoItem` (このタグがあると警告を出さず終了)
- **注文進むボタン**: `.Button--NextStepLogin`
- **見積ボタン**: `#estimate`
- **価格表示エリア**: `.NextStepPriceValue .Price`

#### ページB: 注文確認
- **注文確定ボタン**: `#checkoutNext__order_submit` および `#order_submit`
- **小計表示**: `#checkoutNext__rg_sub_total_span`
- **配送料表示**: `#checkoutNext__rg_d_charge`
- **支払方法ラジオボタン**: `.pay_method`
- **特定の行の非表示**: `tr[data-analytics-tag="code_part"]`

#### ページC: 見積書
- **価格エリア**: `.kakaku`

### 3. スタイル定義
- 無効化時のスタイル（`is-Disabled` クラス）や警告メッセージのスタイルは `applyRestriction` 関数内に定義されています。

## 引き継ぎ・運用上の注意

- **パニックモードが起動した場合**: 
  - まず「カートが空」でないか確認してください。
  - カートに商品があるのにロックされる場合は、上記「セレクタ一覧」のいずれかがサイト側で変更された可能性が高いです。開発者ツールで新しいIDやクラス名を調査し、コードを更新してください。
- **税計算ロジック**: 
  - `checkTaxDecimal` は整数の乗算で小数点第一位を判定しており、グローバルの `TAX_RATE` に連動しています。税率変更時は `TAX_RATE` を変更するだけで計算ロジックにも反映されます。
- **見積書の注釈**: 
  - ページCの下部には、拡張機能によってチェックされた旨の注釈が自動挿入されます。これもサイト構造が変わると挿入位置がずれる可能性があります。
